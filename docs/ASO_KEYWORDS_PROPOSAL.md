# App Storeページ キーワード改善提案（v1.1提出に同乗）

2026-07-21のSERP調査に基づく提案。**「城 記録」「城 アプリ」のGoogle SERPでは競合アプリのApp Storeページが直接1ページ目にランクインしている**（例: リリース6日目の「お城マップ」、「お城がいいね」はサブタイトルの「記録が残せる」で「城 記録」派生クエリに出現）。つまりApp Storeページのメタデータ改善は、ASO（ストア内検索）とGoogle SEOの両方に効く一石二鳥の施策。

実作業はApp Store Connect（shirodoリポ側のリリースフロー）。現行の正確なタイトル・サブタイトル・キーワード欄はApp Store Connectで要確認のうえ、以下と差分を取ること。

## 提案内容

| 項目 | 提案 | 狙い |
|---|---|---|
| タイトル（30字以内） | `城道（しろどう）- 城めぐり記録` | ブランド＋最重要キーワード「城めぐり記録」を直結。Google SERPでのタイトル表示にもそのまま乗る |
| サブタイトル（30字以内） | `日本100名城の登城記録・写真・地図` | 「100名城」「登城記録」を補完。競合「お城がいいね」がサブタイトルの「記録が残せる」でGoogle流入を取っている実績パターンの踏襲 |
| キーワード欄（100字） | `城,城めぐり,登城,記録,日本100名城,スタンプ,御城印,城巡り,歴史,天守,城郭,旅` | ストア内検索用。「城 記録」「100名城 アプリ」系の複合検索をカバー |
| プロモーションテキスト | 「広告なし・登録不要。毎朝ひとつの『問い』で城の文脈を学び、登城を記録する」 | 審査不要でいつでも変更可。差別化要素を前面に |

## 注意
- タイトル変更は審査対象。v1.1の再提出時に同乗させるのが効率的（単独更新は不要）
- 「スタンプラリー」を名乗るのは避ける（公式スタンプラリーとの誤認リスク。LP側FAQでも「併用できる」という整理で統一済み）
- 変更後、LP側のJSON-LD（MobileApplication.name）とalternateNameの整合を確認すること（現在: `城道（SHIRODO）`＋alternateName配列）
- 効果測定: App Store Connectの「App Storeブラウズ/検索」ソース別DL＋GSCの「城 記録」系表示回数（現在0）を4週間観測
